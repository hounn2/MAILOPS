# 智能问答自动回复功能使用说明

## 🎯 功能概述

智能问答功能可以根据邮件内容，自动从问答库中模糊匹配问题，并按问答库中的标准答案进行回复。

**适用场景：**
- 客户常见问题自动回复
- 售前咨询自动应答
- 售后服务自动处理
- 减少人工客服工作量

## 📁 文件说明

```
MAILOPS/
├── qa_database.json          # 问答库配置文件 ⭐
├── qa_engine.py              # 智能问答引擎
├── outlook_assistant_win_standalone.py  # 已集成问答功能
└── config.json               # 包含问答规则配置
```

## 🚀 快速开始

### 1. 配置问答库

编辑 `qa_database.json` 文件：

```json
{
  "qa_pairs": [
    {
      "id": "qa_001",
      "questions": [
        "产品价格是多少",
        "多少钱",
        "怎么收费"
      ],
      "answer": "您好！我们的产品价格：\n基础版：¥999/年\n专业版：¥2999/年",
      "similarity_threshold": 0.6
    }
  ],
  "settings": {
    "default_similarity_threshold": 0.6,
    "include_unmatched_notice": true,
    "unmatched_notice": "您好！您的问题我已收到，我会尽快为您处理。"
  }
}
```

### 2. 启用智能问答规则

编辑 `config.json`，启用 `rule_005`（智能问答自动回复）：

```json
{
  "id": "rule_005",
  "name": "智能问答自动回复",
  "enabled": true,  // 改为 true 启用
  "conditions": {
    "match_all": true,
    "items": [
      {
        "field": "subject",
        "operator": "contains",
        "value": ["咨询", "问题", "价格", "发货"]
      }
    ]
  },
  "actions": [
    {
      "type": "qa_reply",
      "similarity_threshold": 0.6,
      "include_original": false,
      "subject": "RE: {original_subject}"
    }
  ]
}
```

### 3. 运行程序

```bash
python outlook_assistant_win_standalone.py --once
```

## 📋 问答库配置详解

### 字段说明

#### qa_pairs（问答对列表）

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | string | 问答对唯一标识 | "qa_001" |
| questions | array | 问题的多种表述方式 | ["多少钱", "价格是多少"] |
| answer | string | 标准答案 | "产品价格为..." |
| similarity_threshold | float | 匹配阈值（0-1） | 0.6 |

#### settings（全局设置）

| 字段 | 类型 | 说明 | 默认值 |
|------|------|------|--------|
| default_similarity_threshold | float | 默认匹配阈值 | 0.6 |
| include_unmatched_notice | bool | 未匹配时是否发送默认回复 | true |
| unmatched_notice | string | 未匹配时的默认回复内容 | "您好..." |

### 匹配阈值建议

- **0.5-0.6**: 宽松匹配，可能匹配到一些不太相关的问题
- **0.6-0.7**: 适中匹配，平衡召回率和准确率（**推荐**）
- **0.7-0.8**: 严格匹配，只匹配高度相似的问题
- **0.8+**: 非常严格，可能漏掉一些正确的问题

## 🔧 动作配置详解

### qa_reply 动作参数

```json
{
  "type": "qa_reply",
  "similarity_threshold": 0.6,    // 可选，覆盖全局阈值
  "include_original": false,       // 可选，是否包含原始邮件
  "subject": "RE: {original_subject}"  // 可选，自定义主题格式
}
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | string | 是 | 固定值 "qa_reply" |
| similarity_threshold | float | 否 | 匹配阈值，默认使用全局设置 |
| include_original | bool | 否 | 是否包含原始邮件内容，默认 false |
| subject | string | 否 | 回复主题模板，默认 "RE: {original_subject}" |

## 🧠 匹配算法说明

### 相似度计算方法

系统使用多种算法综合计算相似度：

1. **SequenceMatcher（40%权重）**
   - 基于最长公共子序列
   - 计算文本整体相似度

2. **Jaccard相似度（30%权重）**
   - 基于词集合交集
   - 计算词汇重叠程度

3. **关键词匹配（20%权重）**
   - 计算匹配关键词比例
   - 重视核心词汇匹配

4. **包含关系（10%权重）**
   - 检查文本包含关系
   - 短文本包含在长尾文本中的加分

**最终相似度 = 0.4×SequenceMatcher + 0.3×Jaccard + 0.2×关键词 + 0.1×包含关系**

### 文本预处理

- 去除所有标点符号
- 去除多余空格
- 转换为小写

## 📝 问答库编写建议

### 1. 问题的多种表述

每个问答对应包含3-5种不同的表述方式：

```json
{
  "questions": [
    "产品价格是多少",
    "多少钱",
    "价格",
    "怎么收费",
    "费用是多少"
  ]
}
```

### 2. 答案格式规范

- 开头使用礼貌用语（"您好！"）
- 使用清晰的段落分隔
- 重要信息用列表展示
- 结尾提供联系方式

示例：
```json
{
  "answer": "您好！\n\n关于退款流程：\n\n1. 申请条件：\n   - 7天无理由退货\n   - 质量问题30天内免费退换\n\n2. 申请方式：\n   - 登录账户 → 我的订单 → 申请退款\n   - 或联系客服：400-xxx-xxxx\n\n如有疑问请随时联系。"
}
```

### 3. 分类整理

按业务场景分类：
- 产品价格类 (qa_price_001, qa_price_002...)
- 发货配送类 (qa_shipping_001...)
- 退款售后类 (qa_refund_001...)
- 技术支持类 (qa_support_001...)

## 📊 性能优化建议

1. **问答对数量**
   - 建议控制在 50-200 条
   - 过多会影响匹配速度
   - 可以按业务分类多个问答库

2. **问题表述**
   - 每个问题提供3-5种表述
   - 覆盖客户常用的表达方式
   - 避免过于相似的问题

3. **阈值调整**
   - 根据实际效果调整阈值
   - 观察日志中的匹配情况
   - 逐步提高匹配准确率

## 🔍 故障排除

### 问题1：所有邮件都不匹配

**原因：**
- 阈值设置过高
- 问答库问题表述不够全面

**解决：**
```json
{
  "similarity_threshold": 0.5  // 降低阈值
}
```

### 问题2：匹配错误的问题

**原因：**
- 阈值设置过低
- 问题表述过于宽泛

**解决：**
```json
{
  "similarity_threshold": 0.7  // 提高阈值
}
```

### 问题3：回复内容为空

**原因：**
- 问答库文件路径错误
- JSON格式错误

**解决：**
1. 检查 `qa_database.json` 是否存在
2. 使用 JSON 格式验证工具检查文件
3. 查看日志文件确认加载情况
