# Outlook邮件助手 - Web控制界面

## 功能特点

✅ **可视化规则管理** - 图形化界面增删改查规则  
✅ **实时执行** - 一键执行邮件处理，查看进度  
✅ **完整日志记录** - SQLite数据库记录每次执行详情  
✅ **统计面板** - 仪表盘展示历史数据和今日统计  
✅ **响应式设计** - 使用Bootstrap 5，支持各种屏幕尺寸

## 文件结构

```
MAILOPS/
├── web_app.py                 # Flask Web主程序 ⭐
├── outlook_assistant_win_standalone.py  # 邮件处理核心
├── templates/
│   └── index.html            # Web界面
├── static/
│   ├── css/
│   │   └── style.css         # 自定义样式
│   └── js/
│       └── app.js            # 前端逻辑
├── start_web.bat             # Windows启动脚本
├── config.json               # 配置文件
└── execution_logs.db         # 执行日志数据库（自动创建）
```

## 快速开始

### 1. 安装依赖

```bash
pip install flask flask-cors pywin32
```

或安装所有依赖：
```bash
pip install -r requirements_web.txt
```

### 2. 启动Web服务

**方法一：直接运行**
```bash
python web_app.py
```

**方法二：使用启动脚本（Windows）**
```bash
start_web.bat
```

### 3. 打开浏览器

访问: http://localhost:5000

## 使用说明

### 仪表盘
- 查看总执行次数、已处理邮件数、匹配规则数、执行动作数
- 今日统计快速概览
- 最近执行记录列表

### 规则管理
- **新建规则**: 点击"新建规则"按钮
- **编辑规则**: 点击规则卡片上的"编辑"按钮
- **删除规则**: 点击规则卡片上的"删除"按钮
- **条件设置**: 支持AND/OR逻辑，多种字段和操作符
- **动作设置**: 回复、转发、移动、标记已读

### 执行记录
- 查看所有历史执行记录
- 点击"详情"查看单次执行的邮件处理详情
- 包括处理时间、匹配数量、执行动作等

### 系统设置
- 修改检查间隔
- 设置是否只处理未读邮件
- 设置每批最大邮件数
- 配置处理后是否标记已读

### 立即执行
- 左侧菜单下方点击"立即执行"按钮
- 可选择"试运行模式"（不实际执行操作，仅测试）

## 技术栈

- **后端**: Python Flask + SQLite

## API接口

### 配置管理
- `GET /api/config` - 获取配置
- `POST /api/config` - 更新配置

### 规则管理
- `GET /api/rules` - 获取所有规则
- `POST /api/rules` - 添加规则
- `PUT /api/rules/{id}` - 更新规则
- `DELETE /api/rules/{id}` - 删除规则

### 执行控制
- `POST /api/execute` - 立即执行邮件处理

### 日志查询
- `GET /api/logs` - 获取执行日志列表
- `GET /api/logs/{id}/details` - 获取某次执行详情

### 统计信息
- `GET /api/statistics` - 获取统计数据
- `GET /api/status` - 获取当前状态

## 数据存储

### 配置文件 (config.json)
- 存储规则和模板配置
- JSON格式，便于手动编辑

### 日志数据库 (execution_logs.db)
- SQLite数据库
- 自动创建，无需手动配置
- 包含两张表：
  - `execution_logs`: 执行记录
  - `email_details`: 邮件处理详情

## 故障排除

### 无法启动
- 检查Python是否安装
- 检查依赖是否安装完整
- 检查端口5000是否被占用

### 页面无法访问
- 确保服务已启动（看到"Running on http://0.0.0.0:5000"）
- 检查防火墙设置
- 尝试使用 http://127.0.0.1:5000 访问

### 规则不生效
- 检查Outlook是否已运行
- 使用"试运行模式"测试
- 查看日志文件 outlook_assistant.log
- 检查规则条件和动作配置

### 数据不显示
- 刷新页面
- 检查浏览器控制台是否有JavaScript错误
- 确保config.json格式正确
